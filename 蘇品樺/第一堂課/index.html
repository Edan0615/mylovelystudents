💡 程式開發心法

「在動手寫程式之前，先武裝你的大腦」

K

K.I.S.S.

Keep It Simple and Stupid

保持簡單明瞭。最好的程式碼是連新同學看了都能秒懂的結構。不要過度設計，能用簡單邏輯解決的問題，就不要寫得太深奧。

🎯 最簡單的解法，通常就是最好的解法
D

D.R.Y.

Don't Repeat Yourself

不要重複你自己。如果同樣的代碼寫了兩次以上，就應該將它模組化。讓程式具備高度的可複用性，這也是展示型網頁的核心靈魂。

⚡ 懶惰是工程師的美德：我們只寫一次，然後重複使用

🏗️ 網頁三本柱

「HTML 為骨、CSS 為皮、JavaScript 為肌」

HTML

網頁的骨架

定義內容的結構，如標題、段落與按鈕。就像是房子的主體結構。

CSS

網頁的皮囊

負責網頁的顏色、排版與動畫。就像是房子的裝潢與設計美感。

JavaScript

網頁的肌肉

提供網頁的互動行為與動態效果。就像是房子的電力與自動化系統。

你的手機或電腦。當你輸入網址，就是在發出「請求 (Request)」。

📱 響應式網頁設計 (RWD)

Responsive Web Design - 让网页在不同装置上都能完美呈现

什麼是 RWD？

同一個網頁程式，根據螢幕寬度自動調整排版。透過 Media Query 或框架格線，讓內容在電腦上橫向、在手機上垂直堆疊。

實戰演練：從零構建一套計算機

一步步觀看 HTML (骨)、CSS (皮)、JavaScript (肌) 如何協作

Phase 01: HTML

建立內容骨架

我們首先用 HTML 定義內容。這時候完全沒有排版與按鈕樣式，只有一堆文字和按鈕散落在頁面上。

<!-- 顯示器 -->
<div class="display">0</div>

<!-- 數字按鈕 -->
<button>1</button> <button>2</button> ...
Phase 02: CSS

賦予視覺皮囊

加入 CSS 後，我們定義了顏色、間距與 Grid 網格佈局。這讓網頁看起來像一個現代化的工具設計。

.calculator { display: grid; }
.display { background: #f8fafc; }
button {
  padding: 1.5rem;
  border-radius: 1rem;
}
Phase 03: JavaScript

注入互動肌肉

最後加入 JavaScript。當按鈕被點擊，程式會捕獲數值並即時反映在畫面上。這時候計算機才真正「活」了過來！

$('.btn').on('click', function() {
  let num = $(this).text();
  // 將數字寫入顯示區
  $('#screen').text(num);
});
