Vue 的演進：世代革命

Vue 2 (Legacy)

使用 Options API。邏輯易分散於各區塊，在大規模專案中複用性較低。

Vue 3 (Current)

引進 Composition API。實現極佳的邏輯解耦，讓代碼更易維護且輕量。

開發語法比較

Options API (古典風格)

邏輯依據 Hook 名稱 (data, methods) 散落在各處，隨專案變大，維護難度明顯上升。

<script>
export default {
  data() {
    return { count: 0 }
  },
  methods: {
    add() {
      this.count++;
    }
  }
}
</script>
Composition API (現代風格)

將相關邏輯依功能「打包」在一起，像是專業模組化工廠，簡潔且具備高度複用性。

<script setup>
import { ref } from 'vue';

const count = ref(0);
const add = () => {
  count.value++;
};
</script>
