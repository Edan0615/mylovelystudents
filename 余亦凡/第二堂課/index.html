掌握 Vue.js 的
響應式指令與組件

在這一堂課中，我們將學習 Vue.js 中最重要的各種指令與組件基礎。這些工具能讓你輕易地操控 DOM，並以此構建出高性能、易維護的前端應用。

核心基石：Props & Ref

為什麼需要 Ref？(響應式的秘密)

在普通 JavaScript 中，改變一個變數的值，瀏覽器是不會知道要更新畫面的。為了讓 Vue 「聽得到」數據的變化，我們必須使用 ref()。

🔍 原理解析：

  • 包裝紙 (Wrapper)： ref 會把你的值包進一個「物件」裡。
  • .value 的由來： 因為它是物件，所以我們在 JS 裡必須透過 xxx.value 來存取。
  • 自動拆解： 在 HTML (Template) 裡，Vue 會自動幫你拆掉包裝，所以不用寫 .value。

Props (組件的單向對話)

想像父組件是「老闆」，子組件是「員工」。老闆透過 Props 下達指令。 原則： 員工只能「看」指令，不能隨便改老闆給的原始文件（單向數據流）。 

// Script: 宣告響應式變數
const count = ref(0); 

// Template: 自動讀取 (免 .value)
<button @click="count++">\{{ count }}</button>

條件渲染：v-if vs v-show

v-if (真實的銷毀)

當條件為假時，元件會被 **從 DOM 中徹底移除**。適合不常切換、或初始渲染開銷大的場景。

v-show (CSS 切換)

元件始終被保留在 DOM 中，只是透過 CSS 的 display: none 來切換。適合 **頻繁切換** 的場景。

列表渲染：v-for

`v-for` 指令基於一個數組來渲染一個列表。語法格式為 `item in items`。

<ul>
  <li v-for="user in users" :key="user.id">
    {{ user.name }}
  </li>
</ul>

效能核心：計算屬性 (Computed)

為什麼它叫「智慧計算」？

`computed` 具有 **Caching (緩存)** 機制。它會記住計算結果，只有在它依賴的數據（例如 money）變動時，才會重新計算。

const money = ref(100);
const level = computed(() => {
  console.log("正在計算會員等級..."); 
  return money.value > 1000 ? '金牌' : '普卡';
});

// Q1: 目前 console 是空的 (沒人讀取，不計算)
// Q2: 執行讀取
console.log(level.value); // 印出 "正在計算會員等級..."
// Q3: 數據沒變，再讀一次
console.log(level.value); // 完全沒動靜！(直接拿快取)
// Q4: 數據變了，但沒人讀取
money.value = 5000;      // 還是沒動靜！(懶加載)
// Q5: 終於有人要看結果了
console.log(level.value); // 才會印出第二次 "正在計算..."

Vue Router 完全解析

❌
無 Router (手動切換)
  • 使用 `v-if` 或 `is` component 來切換視圖。
  • 瀏覽器 URL 不會改變，無法使用「上一頁/下一頁」。
  • 難以分享特定頁面的網址。
  • 適合：小型單一組件、浮窗、或不需要深度鏈接的工具。
✅
有 Vue Router
  • 真正的單頁面應用 (SPA) 體驗。
  • URL 自動同步，導航完全由 Vue 掌控。
  • 支持嵌套路由、路由守衛、動態參數。
  • 適合：中大型應用、CMS、部落格、任何需要導航的網站。
