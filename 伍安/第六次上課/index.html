01

開發者的「主控權」與「AI 溝通術」

停、看、聽：先有架構，後有代碼

寫程式最困難的不是文法，而是「邏輯與工具的選型」。看到需求時，不要馬上打開 IDE。先停下來，在腦中跑一遍邏輯。

「框架是為了解決複雜問題，
而不是為了讓簡單問題變得複雜。」

工具選型的自問自答

我現在要做的是一個簡單的活動頁，還是需要複雜狀態管理的後台？

真的需要動用到 Vue 這種大型框架嗎？還是簡單的 HTML/JS 就能解決？

先動腦，再動手 (Mind over Code)

在紙上畫出 Data Flow。如果連妳都講不清楚數據是怎麼轉彎的，AI 給出的代碼也只會是一堆會跑的垃圾。

結構化思維 (Computational Thinking)

AI 擅長寫程式碼，但不擅長「拆解問題」。在問 AI 之前，妳必須先能用中文描述出邏輯流程。

  • 步序一：點擊按鈕後觸發事件
  • 步序二：檢查欄位是否符合格式
  • 步序三：發送 API 請求並獲取回應

Context (上下文) 的重要性

如果不告訴 AI 妳是用 Vue 3、Vite 還是 Tailwind，它可能會給妳過時或不相容的答案。

// 💡 好的 Prompt 應該包含環境資訊：
"我正在使用 Vue 3 (Composition API) 和 Tailwind CSS...
請幫我寫一個響應式的導航欄組件。"

警惕 AI 的「違章建築」心理

AI 的解決邏輯往往是「增量式解決」。當妳指出錯誤，它傾向於在原本錯誤的基礎上疊加補丁，導致程式碼變得臃腫難維護。

主動斷捨離

果斷要求重寫：「這太複雜了，我們能不能重啟，用更簡潔的邏輯重新構思？」

模組化控制

功能拆細，每次只讓 AI 處理一小塊邏輯。防止它往上無限疊加「違章代碼」。

避免「重造輪子」的 AI 溝通術

AI 有時因為不知道妳已經裝了什麼，會幫妳寫出一堆重複的功能。要讓 AI 為妳工作，必須先教它認識妳的工具箱。

實例 A：清單同步

在開始對話時直接定義範圍：「我的專案目前使用了 **Vue 3, Tailwind CSS, 和 Pinia**。請在不增加新套件的前提下，實現以下功能。」

實例 B：組件複用

如果專案裡已有常用組件：「請參考我現有的 **BaseButton.vue** 組件來產出新頁面」，而不是讓它重新寫原始標籤。

❌ 錯誤的 Debug
「這裡報錯了，幫我修好。」
(AI 可能會各種亂猜、亂補補丁)
✅ 正確：The Path of Doubt
「這裡報錯了。我懷疑是 A 傳給 B 的參數不對，或者是 C 的非同步還沒完成。請幫我檢查這條路徑。」

這不僅縮小了 AI 的搜索範圍，更能強迫妳思考資料流 (Data Flow) 到底是怎麼跑的。

「文件檢索」與「AI 生成」的混合技能

這不是二選一，而是 1+1 > 2 的協同作業。

維度 查閱 Documentation (傳統) 使用 AI 輔助 (現代)
優點 資訊絕對準確、結構完整、版本明確。 速度快、直接給出範例、符合特定情境。
缺點 閱讀量大、需要時間尋找對應章節。 可能有幻覺、代碼品質參差不齊。
教學策略 用來建立基礎認知：了解框架生命週期、核心概念。 用來快速實作：處理瑣碎邏輯、產出樣板代碼。
02

接案與職涯：從代碼到商業價值

接案身份：個人獨立 vs. 成立工作室

當妳開始接案，首先要面臨的是「身份」的選擇，這涉及到法律責任與稅務負擔：

個人名義接案
  • 靈活度高，適合小規模或兼職營運。
  • 無法開立正式發票，大型企業合作受限。
成立工作室 / 公司
  • 專業形象佳，可開立發票進行正式商業採購。
  • 稅務規劃更有彈性，利於長期經營。

挑選客戶與避雷：妳沒時間處理溝通內耗

建議年齡層：與妳相仿 (20s)

與年長者 (70-80歲) 溝通往往存在嚴重的 **「科技代溝」**。對他們而言，連「登入」可能都是障礙，這會大幅增加妳的非開發成本。挑選熟悉網路使用習慣的客戶（如同齡層），能讓協作效率翻倍。

三大核心紅燈 (Red Flags)
  • 「這很簡單，就跟 FB 社團一樣。」(危險信號：完全不理解開發複雜度)
  • 「我隨便找都能找人做，是給妳機會。」(PUA 型：貶低價值，逃避公平收費)
  • 對於客戶背景不透明、甚至從事不正當行業。(一定要挑選正經經營的客戶)
理工人的角色轉型：翻譯官 (Translator)

客戶不會懂 "RGBA" 或 "API Endpoint"。妳的責任是將技術細節翻譯成「商業結果」。

如果妳只會講數據，而客戶只會講感覺，專案將永遠陷在溝通泥淖。記住：**聽懂客戶沒說出口的需求，才是接案高手的展現。**

案源攻略：妳的客戶在哪裡？

Sources of Projects

FB 接案社團

案量大，但品質不一。需具備高度篩選客戶的眼力。

Task 123

台灣在地老牌平台。適合新手建立早期評價與作品集。

Upwork / Int'l

國際頂尖戰場。單價最高，適合對英文與技術有自信者。

親友介紹

信任度最高。考慮與「業務型夥伴」合作，專職開發。

40% / 40% / 20%
專業防禦性收費模式

保護妳的勞力價值，確保專案全程具備健康的現金流與開發動力：

40%

訂金 (Deposit)

合約簽訂後立即收費。這代表雙方的協作誠意，沒收到這筆錢絕不動工。

40%

中款 (Kick-off)

階段性驗收（如 UI 設計確認、核心模組完成）後收取。維持長跑開發的動力。

20%

尾款 (Acceptance)

專案結案、交接後 7 日內收取。包含後續維護 (Warranty) 的啟動。

妳賣的是「解決方案」
而非幾行代碼。

案例分析：市面上有人靠一個不到 100 行的 HTML 表格就收費 10 萬，累積甚至達到 40 萬。因為這張表格解決了客戶原本需要手動處理、每月虧損數萬元的流程痛點。

AI 時代，代碼生成不再昂貴。真正的價值在於妳能否推銷自己，讓客戶相信妳能「聽懂需求」並「解決問題」。

Mindset Shift: 學術 vs 接案
學術思維

「這技術太基礎，收這麼多錢良心難安。」

接案思維

「解決這問題能幫客戶省下每年百萬勞力。對他而言，收 10 萬太划算了。」

「一條龍」
的開發者洗禮。

在獨立接案中，妳必須扛起 UI、邏輯與部署。這能強迫資工系學生跳出 IDE，理解產品的「全貌」。

前端 UI/UX 後端 API 雲端部署
有效溝通術

Rule 01

不要聊技術，聊「目標與成果」。

Rule 02

流程透明化。定期週報是建立信任的關鍵。

Rule 03

提早交付草圖 (Lo-fi)，避免全盤推翻。
