在設計網頁上 了解網站的類別是非常重要的 因為
1.一方面可以讓你知道你所需要的時間 (可以做時間上的安排)
2.一方面可以讓你知道這個東西需要收多少錢(越複雜越貴)
3.一方面可以了解這樣類型的網站 大概會有怎樣的結構

網頁的類別大致上可以分成:
1.展示型網頁
特性： 主要是給人看的，像是一本精美的電子雜誌。
技術重點： 強調視覺設計、排版、美感。
對應邏輯： 幾乎沒有判斷式，頂多是「點擊按鈕跳轉頁面」。
例子： 企業官網、活動報名頁、個人作品集。
對學生的意義： 這是最容易上手的接案類型，重點在於讓使用者「心動」。

2. 前端邏輯型網頁 (Client-side Logic)
特性： 像是一個隨身工具。不需要跟伺服器溝通太深，但要在瀏覽器上完成運算。
技術重點： 互動性、狀態管理（比如：按了 A，畫面 B 要跟著變）。
對應邏輯： 大量的 if-else。例如（簡易計算機）： if 按下的是數字 $\rightarrow$ 顯示在畫面上；if 按下的是等號 $\rightarrow$ 執行運算。
例子： 房貸試算器、BMI 計算機、簡易的小遊戲。
對學生的意義： 這裡開始考驗設計師對「狀態」的想像力（例如：計算錯誤時長怎樣？）。

3. 全端/後端邏輯型網頁 (Full-stack / Dynamic App)

特性： 是一個活的系統。會存取使用者的資料，並根據資料內容呈現不同的結果。
技術重點： 資料庫架構、權限管理、資訊安全性。

對應邏輯： 複雜的條件判斷。例如（電商網站）： if 庫存 < 0 $\rightarrow$ 顯示完售按鈕；if 使用者是會員 $\rightarrow$ 套用折扣價。

例子： 蝦皮購物、Facebook、銀行系統。

對學生的意義： 這是難度最高也最值錢的類型。設計時必須考慮到「資料從哪來」以及「各種極端情況」。
