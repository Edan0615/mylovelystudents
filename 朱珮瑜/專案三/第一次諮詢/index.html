Core Philosophy

寫程式最核心的是「想」，而非「打字」

什麼是 Vibe Coding？

在這個由大語言模型主導的新時代，寫程式不再意味著背誦 API 或通宵 debug。Vibe Coding 是一種全新的開發方式——你負責思考系統的邏輯、資料庫結構與美學脈絡；而枯燥、重複的程式碼撰寫與基本的 Bug 排除，則全面交給 AI 來執行。

這使你能將精力專注於「網站的靈魂」與「商業邏輯」，而非迷失在括號和分號的細節裡。

為什麼我們要這樣設計課程？

以 Laravel 學校為基底：其實我們會整體蓋在 Laravel 上面，它就像是一個學校一樣。正如我們上面解構的，每個元件都有它特別的職責。我們就是要熟悉這個「學校般」的系統，並且熟練地操控它，來達成我們的所有開發目的。

奠定長遠架構思維：懂得看懂架構，未來想更動任何細節，或是推出 2.0 版本，你都能自己精準找到對應的區塊進行更新與調整。

降低程式碼排斥感：在學習的初期，不需要為了遺漏括號或型別宣告而沮喪，先透過 AI 把網站完整呈現，看見成果，建立信心。

Our Methodology

我們的獨家學習方式

擺脫傳統程式開發課程的沉悶與死記，我們用更高級、更符合大腦直覺的模式來設計這堂課。

A. 用網站技術開發「動態課本」

我們不使用冰冷的 PDF 或傳統簡報。這份課本本身就是使用「網頁技術」打造的高級動態網頁。在閱讀學習的過程中，你就在親自體驗 HTML、CSS、JS 與 jQuery 帶來的流暢互動成果，這本身就是最生動的實戰展示。

B. 生活化接地氣的「概念故事」

我深信再複雜的理工或計算機概念，都可以被說成故事書或極其接地氣的生活場景（如後續的學校部門分工）。我們絕不跟你聊生硬的技術官話，用你聽得懂的語言傳授，但同時保留對應的英文專業術語，方便你日後上網搜尋查閱。

C. 8年計算機體系的「實戰精華」

我會將自己將近 8 年的計算機體系專業訓練、以及在業界積累的實戰開發經驗提煉成課程核心。我不跟你聊虛無飄渺的架空理論，而是直接在實戰中指引你「該怎麼做、怎麼設定、怎麼規避錯誤」，用最少時間傳授最精華的實戰解法。

Learning Risks

如果不像我這樣學，會發生什麼事？

傳統的寫程式學習，往往需要耗費大量的時間去死記與熟練語法（Syntax）等細節。在合適的引導下，這個探索語法的過程自然是非常有趣的事情，如果有機會與閒暇，我也非常推薦去體驗。

然而在現實生活中，坦白說，大家學習程式的核心動力之一是「經濟利益與賺錢」——這完全符合經濟學的供需法則。與學校的學術教育不同，學校可能會希望你從零去做出一個電腦編譯器（Compiler）來證明學術理解；但現實商業市場需要的，是你能快速做出一個 A 產品（APP），去解決 B 問題。這就是為什麼我們不繞遠路。如果你堅持使用傳統死記語法的路徑，你將面臨以下四個危機：

A

自主摸索極度耗時且困難

雖然自主摸索的過程很有趣（我自己也很喜歡），但若沒有一套系統化的地圖做指引，你很容易耗費巨大的時間與心力，甚至在不知不覺中，略微跟不上這個時代急速變化的步伐。

B

被科技的更新速度 Overwhelmed 而放棄

今天的科技更新是以天為單位在推進。若用舊思維一行行背程式碼，你會被成千上萬層出不窮的新語法、新框架徹底 overwhelming（淹沒），最終導致挫折感過重而不想再學。

C

不解底層架構導致無法與 AI 溝通，無法更新線上線下版

AI 的能力取決於你的指令（Prompt）。如果不了解網頁底層的運作結構，你根本無法跟 AI 進行有效溝通。當未來你需要更新、除錯，或是要把程式從本地端（線下版）部署到雲端主機（線上版）時，你將會卡死在架構斷層中。

D

抓不到系統核心，沒辦法「打蛇打七寸」

軟體開發最怕頭痛醫頭、腳痛醫腳。如果不理解整個網站（學校）的分工脈絡，遇到 Bug 時你將盲目打轉，無法精準抓住痛點、做到「打蛇打七寸」，卡在小錯誤上消耗所有學習熱情。

Course Expectations

學完這套技術，你能做到什麼？

這是一門兼具思維高度與工程實戰的全端課程。我們的目標並非讓你死記語法，而是培養你成為能夠在 AI 時代下達正確指令、掌控大局的系統架構者。

熟悉網站前後端開發概念

你將不再覺得網站背後是個看不懂的黑盒子。你會明白資料庫（總務處）、API（通訊橋樑）與前端瀏覽器（諮商室）之間究竟是如何流動、交接與渲染的，徹底打通開發思維。

具備與工程師溝通的語言

未來不論是自己發包接案、創業還是主導公司軟體專案，你都能精準使用專業術語（API 串接、資料庫欄位關聯、路由調配）跟工程師對話，不再因為資訊不對等而有溝通障礙。

理論上可開發任何想像的 APP

Laravel + Vue 是一組沒有上限的底層全端組合。只要掌握了狀態管理與 API 資料交換，理論上人類所能想像得到的任何 APP/網站，你都有能力在 AI 輔助下將其實作出來。

⚠️ 誠實學習提醒與時間規劃

必須特別注意的是，十堂課的時間對於龐大的全端系統來說其實是非常不夠的。因此在課程中，很多較為深奧的理論或底層細節，我們會以「簡單帶過」或「挑選最核心重點講」的方式進行，但我們會確保盡一切努力帶你學到所有該學的核心拼圖。

這是一門最複雜也最難的課程，這也是為什麼我們極力推薦上課時間為「一個禮拜一次」。一週的時間能讓你在課後有足夠的緩衝，去消化複雜的系統邏輯、熟悉這套 Laravel 「學校系統」，並有足夠時間讓你在實作中透過 AI 進行嘗試。

Curriculum Outline

10堂系統化課程規劃

學習路徑：我們會從「後端 ➜ 前端」往上學習

以心理學網站為藍本，我們將由後端往前端逐層往上建造（Backend-first, Frontend-up）。先搭好資料庫地基、串接 API 路由，再構築前端互動與樣式，這是最符合全端科學邏輯的學習路徑。

觀念學習核心

深入理解網頁開發的發展歷史（從早期靜態 HTML、CGI、到動態 MVC 架構）。分析為什麼 PHP 依然是當今全球網頁市佔率最高的語言，以及 Laravel 作為現代化 Web 框架的優雅之處與優勢定位。

實作任務

在本機配置 PHP 開發環境，安裝 Composer 與 Node.js，初探 Laravel 專案的初始目錄結構（瞭解各個資料夾的職責分工），並透過 Git 建立初始版本控制。

觀念學習核心

學習脫離對 GUI（圖形化介面）的依賴，理解終端機（Terminal）與 CLI 指令的運作邏輯。認識 Laravel 內建的強力助理 —— Artisan 命令行工具，它將扮演自動化生成代碼與管理系統的「魔杖」角色。

實作任務

熟練使用 `php artisan` 核心命令，包括專案服務啟動（serve）、自動生成 Controller / Model 架構代碼、以及列出所有系統路由表（route:list），體驗純鍵盤高效率開發。

觀念學習核心

深入「校長室」路由（`routes/web.php`）與「教務處」控制器（Controller）的運作邏輯。理解 HTTP 請求如何被解析分發，並渲染 Laravel 專屬的 Blade 視圖引擎（View），實現最基礎的伺服器端網頁渲染。

實作任務

設定第一個自訂網址路由，撰寫 Controller 控制邏輯，並透過 Blade 模板語法（如 `@if`、`@foreach` 迴圈）將後端資料傳遞並動態呈現在瀏覽器畫面上。

觀念學習核心

擺脫傳統複雜 SQL 語法的硬背。理解什麼是關聯式資料庫、Migrations（資料表版本控制）與 Eloquent ORM（將資料表包裝成物件）。設計心理測驗專案中的「題目、選項、結果」資料關聯。

實作任務

使用 Migration 指令定義 `quizzes`、`questions`、`options` 等資料表結構與欄位，執行資料庫遷移，並在 Model 中設定「一對多」的 Eloquent 關聯，實現物件化資料撈取。

觀念學習核心

體會高級架構的設計思維。隨著功能變複雜，Controller 會變得無比臃腫難以維護。學習「高內聚、低耦合」原則，引進 Service Pattern，將複雜的商業邏輯（如心理測驗的複雜計分演算、分析邏輯）獨立封裝。

實作任務

建立 `QuizService`，把原本塞在 Controller 的測驗分析演算法移至服務層處理，使 Controller 僅負責接收請求與回傳結果，實現極致乾淨且易擴充的後端架構。

觀念學習核心

學習「不重複造輪子」的現代軟體工程思維。理解後端套件管理器 Composer (PHP) 與前端套件管理器 npm / Node.js (JS) 的運作本質與依賴檔案（json/lock 文件），認識靜態資源打包編譯（Vite）。

實作任務

學習使用 `composer require` 安裝後端輔助套件，並使用 `npm install` 配置前端環境，運行 Vite 編譯指令，完成前後端靜態檔案資源的整合建置。

觀念學習核心

專門為「前後端對話」設計的關鍵課程。理解瀏覽器 JavaScript 與伺服器 PHP 之間的溝通管道。深入 HTTP 協議中的 GET/POST 請求、RESTful API 規劃，以及輕量化的資料交換媒介 —— JSON 格式。

實作任務

在 Laravel 中開啟專屬 API 路由（`routes/api.php`），寫出回傳 JSON 的心理測驗 API；並使用前端 JavaScript 工具發送非同步請求，成功讀取並顯示後端提供的資料。

觀念學習核心

壓縮整合 HTML、CSS 與 CSS 框架。學習響應式網頁設計（RWD）的核心「網格系統（Grid System）」，並理解如何用 Bootstrap 提供的現成元件，在完全不寫自訂 CSS 下快速完成後台排版。

實作任務

使用 Blade 引入 Bootstrap 5 CDN，繪製出專業漂亮的後台測驗題目管理列表（CRUD 介面）、新增題目的表單，以及在手機和電腦上都能流暢操作的自適應佈局。

觀念學習核心

若每答一題就要整頁重整，用戶的心理測驗體驗會非常糟糕。壓縮前端 MVVM 概念，認識 Vue 3 的「響應式狀態（Reactive State）」，理解如何讓資料與網頁畫面即時雙向同步。

實作任務

在 Blade 頁面引入 Vue 3，接管心理測驗做題區塊。實作「點擊選項即時切換下一題」、計算作答分數，並在點完最後一題時發送 API 請求給後端。

觀念學習核心

回顧整個專案生命週期與架構。在 Vibe Coding 模式下，當系統出錯時，如何有邏輯地排除 bug？並理解網頁從本機（Local）部署至雲端主機（Production）的生命週期概念。

實作任務

完美串聯資料庫、Laravel 後端、Service 邏輯層、Vue 前端與 API 對話。講師故意設計常見報錯，引導學生透過 Console 與日誌引導 AI 完成精確除錯，並實行專案最終功能總驗收。

Core Frameworks

本專案三大開發框架

我們在專案中將主要使用 Laravel 作為後端基底、Vue.js 作為前端高互動引擎，並以 Bootstrap 作為後台與部分組件的版面樣式。

Laravel

Backend Framework (PHP)

優雅的 PHP 全端框架。它提供了資料庫遷移（Migration）、強大的 Eloquent ORM（用物件的方式管理資料）、內建的安全驗證與路由機制。它將扮演我們網站的「後端心臟」，處理資料運算、存儲與身分驗證。

前往 Laravel 官網

Vue.js

Frontend Framework (JS)

漸進式 JavaScript 框架。它的核心特色是「雙向資料綁定」與「組件化」。當你的心理測驗在前端點擊下一題時，Vue 會即時更新畫面狀態而不需重新整理網頁，讓使用者體驗無比絲滑、沉浸。

前往 Vue.js 官網

Bootstrap

UI/CSS Framework

最受歡迎的 CSS 框架。它預先設計好了大量的網頁元件樣式（如按鈕、輸入框、導航列、表格、網格），能讓我們在幾乎不寫自訂 CSS 的情況下，快速建構出排版整齊、能在手機與電腦上自適應顯示的「後台管理介面」。
