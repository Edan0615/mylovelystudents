01. 核心觀念：同步與非同步的本質

同步 (Synchronous)

核心詞：阻塞 (Blocking)

👤
Wait...
🛑

為什麼它會「卡死」？

JavaScript 是「單執行緒 (Single Thread)」的語言，意思是它一次只能處理一件事。在 同步模式 下，程式碼按順序執行。當執行到「需要時間」的工作（如連線 PHP 或大量計算）時，整個 JS 引擎會停在那裡等待結果。

比喻：就像單線道。前方一台老爺車壞了，後方所有的超跑、公車全都得停下來，直到老爺車開走為止。這就是「阻塞」。

非同步 (Asynchronous)

核心詞：非阻塞 (Non-blocking)

👤
Working!
📱

為什麼它比較「滑順」？

在 非同步模式 下，當執行到耗時工作時，JS 會把這項工作交給瀏覽器的後台處理（Web APIs），然後馬上跳到下一行代碼繼續往下跑。等後台處理完後，再透過「回呼函數」通知 JS 回頭處理剛才的結果。

比喻：就像美食街點餐拿呼叫器。點完餐後，你的「人」是自由的，可以先去找位子、去買飲料甚至去廁所。餐好了器會響，你再去拿。這就是「非阻塞」。

你什麼時候用 jQuery？

當你的老網站已經載入 jQuery 時。它非常穩定，且不需要額外安裝東西。

你什麼時候用 Fetch？

當你寫的是小型、甚至不依賴套件的原生網頁時。它是未來瀏覽器的共同語言。

你什麼時候用 Axios？

現在！所有的 React/Vue 專案。它強大的「攔截器」可以幫你自動在每封信夾帶 Token。

術語大解密：課後必備觀念

🤝

Promise (非同步物件)

Promise 是「未來」的結果。 它是 JS 用來處理「現在還沒好，等等才會好」的工作物件。你可以把它想成披薩店的號碼牌，拿到號碼牌（Promise）代表你未來會得到披薩（資料）。

• Fulfilled：成功交貨 (拿到資料)

• Rejected：交貨失敗 (發生錯誤)

📦

JSON vs XML

資料的打包方式。 XML 是舊時代的笨重包裝（像 HTML 有標籤）；JSON 是現代流行的輕量包裝，傳輸速度快了非常多。

進階對照觀念：catch 與 await 的秘密

!

try...catch：程式的「安全氣囊」

這跟 PHP 的 try...catch 邏輯完全一樣！它是為了防止程式因為一點小事（例如網路斷了、伺服器掛了）就整個崩潰關閉。

JavaScript (前端) PHP (後端) 
try {
  // 執行 API 請求
} catch (err) {
  // 處理失敗
}
<?php
try {
  // 資料庫連線
} catch (Exception $e) {
  // 顯示錯誤
}
?>

※ 重點：兩者的目的都是「優雅地處理失敗」，不讓使用者看到冷冰冰的報錯頁面。

💡

為什麼 Axios 的 catch 比較聰明？

● Fetch API 的「雷點」

即使伺服器回傳 404 (找不到) 或 500 (掛了)，Fetch 依然會認為「我有連到伺服器」，所以會走進 .then() 或繼續執行，你必須手動判斷。

if (!res.ok) { throw new Error('壞掉了'); }

● Axios 的「貼心處」

Axios 會幫你檢查 HTTP 狀態碼。只要 不是 200~299 (例如 404/500)，它就會自覺地「主動彈射」到 catch 區塊。

  • ✅ 邏輯清晰：try 裡面只有「成功」
  • ✅ 安心開發：失敗的事情統一交給 catch

? 概念釐清：Promise 與 await 的關係

Concept / Object

Promise

它是一個「代名詞」。代表一個尚未完成，但未來會好的工作結果。

🎟️
就像飲料店的「取餐號碼牌」。
你拿著它，代表你「將會」得到飲料。
Action / Keyword

await

它是一個「指令」。告訴 JavaScript：除非拿到結果，否則不准執行下一行。

✋
就像你在「櫃檯前持續站著等」。
直到號碼牌變成了真正的飲料。

結論： 請求是由 axios.get() 發起的，而 await 的作用是把你手上的號碼牌轉化為實際數據。

Vue 3 x Axios：實戰模組化

既然我們正在學 Vue，那就要用 Vue 的方式來與 API 溝通。

?!

Wait! await 到底是什麼？

「await 不是 Promise，它是 Promise 的拆封機。」

1. Promise： 包裹。雖然你知道裡面有披薩，但你得用 .then() 苦等拆封。
2. await： 魔法。它讓 JS 停在這一行，直到 Promise 好了並直接幫你「拆開包裹」拿出資料。

※ 這就是所謂的「語法糖」：讓非同步寫起來像同步一樣好讀。

Vue 開發核心三部曲

  • A 建立響應式狀態 (ref)：用來存 API 拿回來的資料。
  • B 生命週期掛載 (onMounted)：網頁一打開就自動去敲 API。
  • C 畫面渲染 (v-for)：將資料倒進 Vue 的卡片模板裡。 
<script setup>
// 1. 準備 Vue 工具箱
import { ref, onMounted } from 'vue';
import axios from 'axios';

// 2. 設定裝資料的籃子
const studentList = ref([]);
const isLoading = ref(true);

// 3. 定義抓資料的大招 (Async/Await)
const getStudents = async () => {
    try {
        // await 會拆開 Promise，直接把內容丟給 res
        const res = await axios.get('api/students.php');
        studentList.value = res.data; // 裝進籃子！
    } catch (err) {
        console.error("API 連線失敗...");
    } finally {
        isLoading.value = false;
    }
};

// 4. 網頁開啟時，自動放大招
onMounted(() => {
    getStudents();
});
</script>
✨ Modern：Async / Await
const get = async () => {
  const res = await axios.get();
  data.value = res.data;
}

優勢： 程式碼呈現垂直排列，看起來就像一般的「同步」指令，維護起來最舒服。

📜 Traditional：.then() Chain
const get = () => { // 這裡沒 async
  axios.get()
    .then(res => { ... })
    .catch(err => { ... });
}

優勢： 適合不需要寫 async 的輕量情境。缺點是程式碼會出現「水平縮排 (鏈條)」，複雜時較難讀。

💡 亦凡小筆記：

在 Vue 裡面用 Axios 時，記得 一定要加 .value (例如 studentList.value = ...)，不然畫面不會跟著更新喔！這點一定要考倒學生。
