深入了解 Bootstrap 框架

快速、響應式、且功能強大的前端開發利器。這次要帶你一窺這個最受歡迎的 CSS 框架的奧秘。

到底什麼是 Bootstrap？

我們在之前的課程中學習過了基本的 CSS，也有簡單介紹過 Bootstrap。但這次，我們要來深入了解它究竟是在做什麼的。

簡單來說，Bootstrap 就是一個讓你更好、更快寫網頁程式的工具包（Framework）。它把全世界前幾頂尖的工程師寫好的 CSS 樣式打包起來，讓你只需要套用他們設定好的名稱，就能直接使用這些漂亮的樣式，大幅度簡化了自己從頭寫 CSS 程式的時間。

寫法的差異：CSS 對決 Bootstrap

來看看如果我們要寫一個「藍色背景、白色文字、有圓角且滑鼠移過去會有變化的按鈕」，兩種寫法有多大的差別：

原版的 CSS 寫法

<!-- HTML 部分 -->
<button class="my-custom-btn">點擊我</button>

<!-- CSS 部分 -->
.my-custom-btn {
    background-color: #0d6efd;
    color: white;
    padding: 10px 20px;
    border: none;
    border-radius: 5px;
    font-size: 16px;
    cursor: pointer;
    transition: background-color 0.3s;
}

.my-custom-btn:hover {
    background-color: #0b5ed7;
}

Bootstrap 的寫法

<!-- HTML 部分 -->
<!-- 只需要加上 btn 和 btn-primary -->
<!-- 完全不需要自己寫 CSS！ -->

<button class="btn btn-primary">
    點擊我
</button>

💡 結論：Bootstrap 讓你免去了撰寫大量 CSS 的麻煩，直接透過 Class 呼叫樣式！

如何開始使用 Bootstrap？

想要了解所有功能，第一步永遠是打開 Bootstrap 官方網站 查看完整技術文件。

引入方式主要有兩種：

1

CDN (內容傳遞網路)

我們這次上課的作法

這是一種最簡單、快速的方式。你不需要下載任何檔案到自己的電腦，只要把 Bootstrap 提供的網址（URL）貼到 HTML 的 <head> 標籤裡面就可以了。

運作原理：當使用者打開網頁時，瀏覽器會根據那段網址，自動去遠端的伺服器抓取 Bootstrap 的 CSS 和 JS 檔案。這很適合初學者和簡單的靜態網頁。

2

npm install

未來學 Vue 框架時常見的作法

npm (Node Package Manager) 是 Node.js 的套件管理工具。透過終端機輸入指令 npm install bootstrap，會把 Bootstrap 真正地「下載」到專案的 node_modules 資料夾中。

運作原理：因為檔案在本地端，我們可以在未來的 Vue 開發中自由地使用 JavaScript 模組系統 (import) 匯入它以進行客製化。不過這個現在太複雜了，我們接下來的課再學！

核心概念：一切都在 Class 裡面

基本寫法邏輯

使用 Bootstrap 幾乎就是在背單字。它的基本寫法就是把所有的設定都寫在 HTML 標籤的 class="..." 屬性裡面。

  • text-center (文字置中)
  • mt-3 (Margin Top 距離上面 3 單位)
  • d-flex (顯示為 Flex 排版)

超豐富的 Components (組件)

除了基礎的排版，Bootstrap 最強大的地方在於它提供了非常多做好的「組件」。你不需要從頭刻一個導覽列或彈出視窗，複製貼上即可！

網頁的骨架：Container 與 Grid System

1 Container (容器)

container 這是排版時最常使用的第一個 class。它最大的好處是在左右兩邊自動留白，把網頁內容居中。

  • 美觀性：內容不會緊貼著左右螢幕邊緣，看起來舒服很多。
  • 實用性：許多網站（如部落格或論壇）兩側的留白部分，正好可以用來放置 Google 廣告 或其他側邊欄資訊。

響應式設計與 Mobile First

Mobile First (手機優先)

如果你仔細看 Bootstrap 的寫法，會發現一個有趣的邏輯：預設的樣式都是給手機看的，當螢幕變大時（透過 sm, md, lg），才去加上新的樣式。 這就是現代網頁開發的核心觀念——Mobile First (手機優先設計)。

效能最好

手機的網路和運算能力通常比電腦差。如果一開始先載入電腦版的肥大程式碼再隱藏起來（以前的作法），對手機來說負擔太大。

邏輯更順

從小螢幕（內容最精簡）往大螢幕（可以展開更多側邊欄和特效）去設計「加法」，比從大螢幕刪減東西（減法設計）來得容易。

💡

歷史小知識：：大約在 2010~2012 年間，智慧型手機開始大爆發，行動裝置的網路流量逐漸超越桌機。也是在那時，Mobile First 成為了業界的標準。Bootstrap 就是這個理念的忠實擁護者。

Bootstrap 的魔法斷點 (Breakpoints)

在現今的時代，網頁在手機、平板、電腦上看都要長得不一樣。這也就是俗稱的 RWD (響應式網頁設計)。 Bootstrap 為了克服這件事，提供了幾個好用的「斷點 (Breakpoints)」。

縮寫 (斷點名稱) 對應裝置 最小寬度條件
(無縮寫, 預設) 超小螢幕 (手機直立) < 576px (Mobile First 預設)
sm 小螢幕 (手機橫放) ≥ 576px
md 中螢幕 (平板電腦) ≥ 768px
lg 大螢幕 (桌上型電腦) ≥ 992px
xl / xxl 超大螢幕 ≥ 1200px / 1400px
運用 Mobile First 思考： d-none d-md-flex
<div class="d-none d-md-flex"> ... </div>
1. 因為是 Mobile First： 最前方的 d-none 會優先作用在手機上，所以這塊區域在手機版會直接消失。
2. 針對大螢幕加上樣式： 當螢幕寬度超過 768px (平板以上) 時，我們才加上附加的 d-md-flex，讓它變成 flex 顯示！

框架的演進：Bootstrap 對決 Tailwind CSS

雖然這堂課我們教的是 Bootstrap，但你可能也聽過另一個現在非常火紅的框架：Tailwind CSS (這個講義本身的排版就是用 Tailwind 寫的！)。讓我們來聊聊它們的差異，以及為什麼現在業界越來越偏好 Tailwind。

📦

Bootstrap (框架感重)

  • ✗ 框架感太重：它提供的是「已經做好的組件」(像是整個卡片、整個按鈕)。這導致你如果不想用它預設的醜醜藍色，要修改起來非常痛苦，必須寫很多額外的 CSS 去覆蓋它。
  • ✗ 客製化困難：怎麼改看起來都還是有「Bootstrap 的味道」。
  • ✓ 優勢：不用思考設計，直接套用就有基礎功能，非常適合後台系統。
🎨

Tailwind CSS (現代、美觀)

  • ✓ 原子化 CSS：它不給你做好的「卡片」，而是給你 p-4(內距)、bg-white(白背景)、rounded-lg(圓角) 這樣的「積木」。
  • ✓ 高度客製化與美觀：因為是用積木拼出來的，所以你可以完全根據自己的美感打造出現代、精緻的 UI，不會受到框架的限制。這也是為什麼現在新創公司與現代網頁幾乎都轉換到了 Tailwind。

未來趨勢：Vibe Coding 的時代

🤖 傳統寫程式 (Classical Coding) 已成過去？

我們今天學了很多 CSS、Bootstrap 或是未來會學的框架。但實話實說：在 Vibe Coding (依賴 AI 生成來寫程式) 的強勢來襲下，整個開發的生態圈已經發生了巨大的改變。

🎓 頂尖學府的震撼彈：史丹佛大學 (Stanford)

史丹佛大學的電腦科學相關課程，已經將我們現在這種「自己親手一行一行敲打程式碼」的學習方式，重新命名為 "Classical Coding" (古典程式設計)。

這意味著什麼？意味著單純敲打程式碼，已經像傳統的手工藝一樣，未來前端甚至大部分後端的「純勞力寫碼」工作，都能夠很大程度地被 AI (像是你之前提到的 AI 自動生成網站) 給取代。

💡 那麼，工程師的價值在哪裡？

既然 AI 都能寫了，我們除了依賴它，在未來的市場中要如何凸顯自己的不可取代性？差異將會體現在以下幾個關鍵能力：

✨ 美學與設計 (Aesthetics)

AI 可以寫出功能正常的網頁，但能做出讓人發出「哇」的靈魂設計與精緻排版，仍然需要人類的品味把關與藝術指導。

🧠 創意與行銷 (Creativity & Marketing)

網頁做出來是要給「人」用的。如何結合行銷邏輯、業務需求與創新點子，規劃出有價值的產品藍圖，這是 AI 無法無中生有的。

⚙️ 核心底層邏輯 (Core Logic)

這也是為什麼我們還是要學這堂課！當 AI 產出的程式碼壞掉，或者發生複雜的安全與架構問題時，如果你不懂「底層原理 (像是 CSS 怎麼運作的、數學邏輯是什麼)」，你連怎麼問 AI、怎麼修復都不知道。我們學寫程式，是為了學會「看懂並駕馭 AI 寫的程式」。
